# 掌握卡片/01. Git与GitHub的区别.md

## 我的理解
Git 是一个本地的工具，用来给代码做“游戏存档”（版本控制）。
GitHub 是一个云端的网站，用来存这些存档，方便大家一起玩。
**它们是可以解绑的**：就像 MP3 文件（Git）和网易云音乐（GitHub）。如果网易云挂了，MP3 只要在本地，依然能听，甚至能用 U 盘拷给别人。

## 🏀 类比挂钩
*导师与我的对话精华*
导师：Git 的核心操作流就像网购：Add 是放进购物车，Commit 是下单（在本地生成订单），Push 是快递发货（才真的同步出去）。
我：我之前混淆了 commit 和 push，认为 commit 就是打包+提交。其实 commit 单纯指打包，push 才是真正的提交。

## 踩过的坑
- **误区**：以为 commit 之后 GitHub 上就有了。
- **真相**：`commit` 只是在本地存档。如果不 `push`，电脑爆炸代码就全没了。
- **误区**：以为 GitHub 挂了就不能工作了。
- **真相**：Git 是分布式的。GitHub 挂了，我们依然可以在本地 commit，甚至可以通过局域网直接传 Git 仓库给同事。

## 源真理定义
> **Git** 是一个分布式版本控制系统 (Distributed Version Control System)，用于跟踪代码更改。
> **GitHub** 是一个基于 Git 的云端托管平台，提供了协作、CI/CD 和社交功能。
