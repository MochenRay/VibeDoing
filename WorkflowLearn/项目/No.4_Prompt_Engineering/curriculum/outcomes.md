## 核心可交付产物

- `prompt brief`
  - 证明能力：把模糊需求变成可执行输入输出约束。
  - 可复用位置：PRD 附件、和研发对齐、面试 case。

- `schema spec`
  - 证明能力：把自由文本收敛成系统可消费的结构。
  - 可复用位置：API 设计、前后端对齐、工具接入说明。

- `tool / function contract`
  - 证明能力：定义工具边界、参数语义和高风险确认机制。
  - 可复用位置：workflow 方案、agent 设计、MCP 设计讨论。

- `generative UI draft`
  - 证明能力：把 AI 输出从“文本答案”升级为“可继续操作的界面”。
  - 可复用位置：AI-native 功能方案、作品集 demo、交互设计讨论。

- `prompt iteration record`
  - 证明能力：展示你不是“拍脑袋写 prompt”，而是按失败模式做迭代。
  - 可复用位置：复盘材料、面试问答、团队知识库。

## 应体现的岗位能力

- 如何把模糊需求变成可控输入输出。
- 如何区分 prompt 问题、schema 问题和工具问题。
- 如何在不确定性下设计 AI-native 交互。
- 如何让 AI 输出能被系统消费，而不是只能被人阅读。

## 与主线课程的关系

- 它支撑 `M2 Prompt / Tool Use`
- 它补强 `M10 Generative UI / AI-native 交互`
- 它也会为后续 `M11-M13` 的产品化与作品集输出提供素材
