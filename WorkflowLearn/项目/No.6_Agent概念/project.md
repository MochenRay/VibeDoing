# No.6_Agent概念

> 最近确认时间：2026-04-16
> 状态：首版稳定内容已迁移

## 用途

本项目重写为 `Agent / Workflow / Harness Engineering` 核心课，服务 `AI 应用 PM + Agent / Workflow PM` 主线。

## 目标

- 建立 Agent 系统的通用认知框架，而不是只学某个框架名词。
- 把 workflow 编排、tool contract、state、observability、恢复能力和组织落地串成一门完整课。
- 为后续的 capstone、产品判断和作品集输出提供方法论底座。

## 范围

- `project.md`
- `handover.md`
- `state.md`
- `migration-notes.md`
- `curriculum/*.md`
- `refresh/*.md`

## 非范围

- 不做具体框架跑分表
- 不把某家厂商的功能开关写进稳定主干
- 不把短期 benchmark 排名或工具榜单写进长期课程正文

## 成功标准

- 能解释 Agent、Workflow、Harness 三者的边界与关系
- 能判断何时用明确编排，何时用 agentic planning
- 能设计可恢复、可观测、可交接的工作流和工具边界
- 能把这些判断翻译成 AI PM 的产品方案、验收和作品
