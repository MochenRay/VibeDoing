# concepts

## 模块 P1：Agent 是什么

- Agent 不是“会聊天的模型”，而是能围绕目标进行规划、调用工具、观察结果和继续行动的系统。
- 对产品经理而言，核心不是是否叫 Agent，而是系统是否具备目标导向、状态推进和任务完成能力。
- Agent 的价值来自“完成任务”，不是来自“生成一段看起来聪明的文本”。

## 模块 P2：Workflow vs Agent Judgment

- `Workflow` 强调显式编排、路径可控、状态可追踪。
- `Agent` 强调在不确定环境中根据观察结果动态判断下一步。
- 产品判断不是二选一，而是把确定路径交给 workflow，把需要判断、探索和例外处理的部分交给 agent。
- 判断依据包括：任务不确定性、错误代价、恢复成本、人工接管点和评估证据。

## 模块 P3：ReAct 与思考-行动-观察循环

- `Reasoning + Acting` 是最基础的 Agent 工作范式。
- 思考、行动、观察构成闭环，决定 Agent 是否能在失败后继续调整。
- 对 PM 来说，重要的是理解这个循环怎样影响延迟、错误率、用户感知和接管方式。

## 模块 P4：Tool Contract 与 MCP

- 工具不是“插件名词”，而是 Agent 能执行的外部动作边界。
- 好的 tool contract 需要输入清晰、输出可预测、错误可处理、权限可控。
- MCP 的稳定价值不是某个宿主的支持状态，而是把工具、资源、提示和上下文交换协议化。
- 工具粒度决定系统是否可调试、可替换、可恢复，也决定权限和审计边界。

## 模块 P5：State / Resume / Artifact

- 长流程任务必须有状态，不然就只能每轮从头开始。
- `state` 记录当前进展，`memory` 记录可复用信息，`artifact` 记录过程产物与交接凭证。
- `resume` 设计要明确检查点、幂等操作、重试策略、超时处理和人工接手条件。
- 对产品而言，状态设计直接决定能否恢复、能否人工接手、能否持续运行。

## 模块 P6：Long-running Harness

- `Harness` 是让长任务稳定跑起来的一整套外层控制：初始化、推进、检查点、回退、收口。
- 长流程任务不能只靠模型“自己会做”，必须设计可恢复的执行协议。
- Harness 要管理任务启动、工具调用、进度 artifact、失败分类、恢复路径和结束清理。
- 对 PM 来说，harness 是把 agent 从 demo 变成可交付系统的关键。

## 模块 P7：Observability 与 Reliability

- 只看最终答案不够，必须能看清工具调用、状态变化、失败位置和恢复轨迹。
- Observability 的目标是让问题可定位、可归因、可复盘，并能支持上线 gate 和后续 eval。
- Reliability 不是“永不出错”，而是“出错后可控、可观测、可恢复”。

## 模块 P8：HITL / Handoff / Multi-agent

- HITL 不是兜底口号，而是明确何时暂停、交给谁、带着什么证据继续。
- Handoff 的核心是角色职责、输入输出、审批点、冲突处理和最终 owner。
- 多 Agent 只在 eval 证明单 agent 或 workflow 无法稳定达标时进入方案。
- 多 Agent 协作的价值不取决于概念新不新，而取决于是否真的降低错误率、延迟或人工成本。
- 真正落地时，组织流程、人工复核、审批点和 SOP 改造往往比模型更重要。

## 模块 P9：Agent Security

- Agent 安全不是只防用户越狱，而是防所有进入上下文和工具链的内容改变系统行为。
- 常见攻击面包括：用户输入、检索文档、网页内容、工具返回、多轮上下文、共享 memory 和外部文件。
- 间接 prompt injection 的风险在于模型可能把不可信数据误当成指令，并通过工具执行高风险动作。
- 稳定防线包括：权限最小化、工具输出净化、参数校验、危险动作人工确认、敏感数据隔离、日志审计和失败回滚。
- PM 的责任是把安全写成产品需求、权限模型、launch gate 和 incident review，而不是只丢给工程实现。

## 模块 P10：Agent Cost Model

- Agent 成本不是一次模型调用费，而是模型推理、工具调用、搜索/检索、重试、状态存储、日志、人工复核和恢复操作的总和。
- 长流程任务的成本会被循环、失败重试和多工具链放大；平均成功成本比单次调用成本更重要。
- Reasoning / thinking budget 会改变复杂 agent 的延迟和 token 成本，不能只按最终回答长度估算。
- HITL 也有成本：人工审核时长、队列等待、交接材料质量和误接管都会影响 ROI。
- 成本模型至少要看：单任务成本、成功率调整后成本、峰值成本、人工成本、监控成本和失败回滚成本。

## 模块 P11：Agentic Retrieval in Agent Systems

- Agentic retrieval 在 agent 系统里是“检索作为行动”，不是固定前置步骤。
- 设计重点是 query budget、停止条件、检索证据、失败归因和 trace，而不是让 agent 无限查下去。
- 如果检索动作会访问外部网页、私有文档或工具返回，必须和 P9 的安全边界一起设计。
