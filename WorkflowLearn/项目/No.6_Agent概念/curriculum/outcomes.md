## 核心可交付产物

- `Agent vs Workflow memo`
  - 证明能力：能在方案评审里解释为什么该上 agent 或不该上。
  - 可复用位置：架构评审、面试 case、作品集开篇。

- `tool contract pack`
  - 证明能力：能把工具边界、风险和确认机制产品化。
  - 可复用位置：MCP / connector 方案、runtime 设计、研发对齐。

- `state / resume scheme`
  - 证明能力：能设计可恢复、可中断、可交接的状态结构。
  - 可复用位置：agent runtime、SOP 改造、事故复盘。

- `long-running harness`
  - 证明能力：能为长流程任务设计 initializer、progress artifact 和清理逻辑。
  - 可复用位置：agent 平台方案、长任务产品设计、面试深挖。

- `observability blueprint`
  - 证明能力：能把 agent 故障从“感觉不对”转成可观测事件和日志。
  - 可复用位置：运行面板、值班手册、上线守门。

- `multi-agent handoff plan`
  - 证明能力：能设计角色边界和 handoff，而不是空喊“多 agent”。
  - 可复用位置：workflow orchestration case、作品集中段。

- `organization rollout note`
  - 证明能力：能把 agent 能力接到真实团队流程和 SOP。
  - 可复用位置：业务落地方案、面试落地题。
