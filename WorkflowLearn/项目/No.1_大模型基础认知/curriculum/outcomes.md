## 对应路线

- `M1` LLM 基础边界
- `M3` Retrieval / Grounding 的前置理解
- `M4-M5` 风险、评估与接管的前置理解

## 核心可交付产物

- `LLM boundary memo`
  - 证明能力：能解释模型为什么会错，而不是只会说模型很强。
  - 可复用位置：作品集开场、面试基础题、跨团队对齐。

- `context budget review`
  - 证明能力：能把 token / context 影响翻译成成本、交互和架构判断。
  - 可复用位置：PRD 附件、架构评审、预算讨论。

- `hallucination risk review`
  - 证明能力：能把幻觉风险翻译成引用、fallback 和人工审核设计。
  - 可复用位置：高风险功能方案、上线评审、面试 case。

- `non-expert explainer`
  - 证明能力：能向业务方清楚解释 LLM、RAG、tool use 的区别。
  - 可复用位置：面试答题、内部分享、作品集附录。

## 应体现的基础能力

- 能解释模型为什么会错，而不是只会描述“模型很强”。
- 能估算 token / context 对成本、交互和架构的影响。
- 能向业务方解释为什么需要 RAG、fallback、引用和人工审核。
- 能区分“随机性问题”“知识边界问题”“检索问题”。
