# B1. 主流模型对比

## 核心框架

选型三维度：**能力 × 约束 × 性价比**

### 选型决策树

```
任务需求 → 有合规/数据安全要求？
    │
    ├─ 是（ToG/金融/医疗）→ 国产模型
    │       └─ 需要私有化？→ 通义千问（开源）
    │
    └─ 否 → 任务复杂度？
            ├─ 复杂推理/代码 → Claude Opus / GPT-4o
            ├─ 超长文档（>200K）→ Gemini 3 Pro
            ├─ 实时信息研究 → Gemini（Deep Research）
            └─ 简单/高并发/成本敏感 → GPT-4o Mini / Sonnet
```

## 模型速览

### 国际模型

| 模型 | 核心优势 | 价格（per 1M）|
|------|---------|-------------|
| Claude Opus 4.5 | 推理顶尖、代码、200K | $5/$25 |
| GPT-4o | 生态完善、多模态 | $2.5/$10 |
| Gemini 3 Pro | 1M 上下文、Deep Research | 按用量 |

### 国产模型

| 模型 | 核心优势 | 价格（per 1M）|
|------|---------|-------------|
| 文心一言 4.5 Turbo | 合规、中文优化、政企客户多 | ¥0.8/¥2 |
| 通义千问 Qwen3-Max | 开源、私有化部署 | ¥2/¥6 |
| 通义千问 Qwen3-Turbo | 便宜快速 | ¥0.3/¥0.6 |
| 豆包 | 256K 长上下文 | ¥0.5/¥2 |
| Kimi | 200K 长上下文、C端体验好 | ¥1/¥3 |

## 💡 验证高光

**ToG 项目选型思路**：
1. 合规优先 → 排除国外模型
2. 按受众/质量分档 → 高质量场景用强模型，低优先级用便宜模型
3. 同档位比价 → 让商务根据体量报价

**回应「为什么不用 ChatGPT」**：
- 先打合规牌：上级审批可能驳回
- 再讲数据安全：政府项目数据不能出境
- 最后讲性能：正常业务国产模型够用，差距不大
