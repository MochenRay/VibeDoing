# 掌握卡片：D2. README

## 🧠 我的理解
- **核心价值**：README 是项目的"接客之道"。Empty README = Dead Project。
- **反脆弱机制**：拒绝"口口相传" (Tribal Knowledge)。新员工入职踩坑的过程，就是完善 README 的最佳时机。把"人的经验"固化为"文档的流程"。

## 🔗 类比挂钩
**"乐高说明书封面"**
- 封面告诉你拼出来是什么（效果图），能不能玩（环境要求）。
- 没了封面，这盒积木就是一堆废塑料。

## 🚫 踩过的坑
- **错误**：信任"人"会去维护文档。
- **真理**：**人一定会忘**。唯一能保证 README 里的命令永远不朽的方法，就是把那行命令写进 **CI (自动化脚本)** 里。如果文档里的命令跑不通，代码就合不进去。这是"文档"的最高境界——**Executable Documentation (可执行文档)**。

## 📖 源真理定义
> **README.md**：项目的门户。它是用户打开仓库看到的第一个文件，承担着"你是谁"、"怎么跑"、"怎么用"的各种解释责任。
