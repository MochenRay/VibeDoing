# 掌握卡片：C2. Project看板

## 🧠 我的理解
- **角色转变**：从 "Human API" (搬运状态) 升级为 "Traffic Control" (监控与规划)。
- **核心价值**：State Automation。让代码的物理状态直接驱动管理的逻辑状态，省去了中间的人工同步环节。

## 🔗 类比挂钩
**"自动驾驶"**
- 以前是手排挡（OmniFocus），不仅要看路还要换挡。
- 现在是自动挡（GitHub Projects），手解放出来去摸方向盘（规划未来）。

## 📖 源真理定义
> **GitHub Projects**：一个跨仓库的、支持自动化流转的项目管理视图。它将离散的 Issues 聚合成可视化的进度表。

## 💡 验证高光
- **上帝视角**：Issues 和 Milestones 是被物理限制在 **单个仓库** 里的（Room Level）。Projects 是 **组织级** 的（Building Level）。
- **实战价值**：当产品涉及 iOS、Android、Backend 多个仓库时，Project 是唯一的聚合管理处。

## 🚫 踩过的坑
- **误区**：以为 Milestone 是总进度表。
- **真相**：Milestone 只能管一个仓库。Project 才能管整个产品线。
