# 掌握卡片：C1. GEMINI.md配置

## 🧠 我的理解
- **核心定位**：不论什么情况下都要让Agent记住的规则，才放到GEMINI.md
- **可配置内容**：沟通风格、代码规范、安全限制、行为偏好、工作流规则
- **结构设计**：用户身份 → 行为规则 → 安全限制 → 特定场景
- **分层披露设计**：
  - GEMINI.md：全局规则，每次都读
  - dev.md：开发领域规则，Agent判断有开发任务才读
  - 项目规则：项目级配置，只在该项目生效
- **维护策略**：反复犯错→固化规则、规则冲突→明确优先级、定期review

## 🔗 类比挂钩
**"三层规则架构"**
```
全局规则（GEMINI.md）
    ↓ 条件引用
领域规则（dev.md / writing.md）
    ↓ 按需加载  
项目规则（项目/规则.md）
```

## ⚠️ 常见误区
- **"规则越多越安全"**：规则太多会挤占上下文空间
- **"所有规则放一起"**：应该分层，按需加载
