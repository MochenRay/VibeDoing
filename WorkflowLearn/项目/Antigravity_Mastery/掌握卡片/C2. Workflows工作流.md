# 掌握卡片：C2. Workflows工作流

## 🧠 我的理解
- **核心定位**：对于需要经常执行且执行内容相对固定的流程，固化为 Workflow
- **位置**：`.agent/workflows/[名称].md`
- **执行模式**：串行逐步确认，`// turbo` 可跳过确认
- **完成标准**：等待命令结果（进程结束+退出码），不是发出就完成
- **创建判断**：高频/复杂/易错 → 值得；一次性/简单 → 不需要
- **与其他配置的区别**：规则→GEMINI.md，流程→Workflow，上下文→项目规则

## 🔗 类比挂钩
**"Vibe Learning 系统就是 Workflow"**
- 导师Agent.md = 教学流程
- 实操Agent.md = 验证流程
- 课程Agent.md = 课程管理流程

## ⚠️ 常见误区
- **"所有操作都做成 Workflow"**：一次性/简单操作不需要
- **"Workflow 只能用斜杠命令触发"**：自然语言也可以

## 💡 验证高光

### 掌握验证：三种需求分类
1. 每天检查项目状态 → **Workflow**（高频+固定步骤）
2. 一次性数据库迁移 → **直接执行**（一次性）
3. 变量驼峰命名 → **GEMINI.md / dev.md**（规则放规则文件）

## 📖 源真理定义
> **Workflows**：可复用操作脚本，定义特定任务的执行步骤。位于 `.agent/workflows/` 目录，使用斜杠命令或自然语言触发。串行执行，可用 turbo 注解跳过确认。
