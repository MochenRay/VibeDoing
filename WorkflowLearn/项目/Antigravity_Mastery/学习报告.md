## 学习路径回顾

### 使用层（A1-A4）✅
**目标**：掌握 Antigravity 的基本使用

| 节点 | 核心要点 |
|-----|---------|
| A1. Agent执行模式 | Agent = 有边界的执行者，PM 要明确目标、设定边界、过程监督 |
| A2. 提示词艺术 🚧 | 目标→约束→方法的层级结构，好的提示词 = 好的需求文档 |
| A3. 任务规划 | 任务粒度分级、依赖管理、并行与串行 |
| A4. 验证与纠错 | 分层验证（即时/阶段/最终）+ 错误快照 |

### 原理层（B1-B4）✅
**目标**：理解 Agent 的工作原理

| 节点 | 核心要点 |
|-----|---------|
| B1. Agent Loop 🚧 | 感知→思考→行动→观察→循环，可控点在四个阶段 |
| B2. Tool Calling | AI 从"顾问"变"执行者"的关键，有风险需管控 |
| B3. Context管理 | 信件结构 + 分层规则体系，GEMINI.md 不会被冲掉 |
| B4. Artifacts系统 | 处处留痕 = 可调试性，三件套：task.md, plan, walkthrough |

### 定制层（C1-C4）✅
**目标**：学会配置和扩展 Agent 能力

| 节点 | 核心要点 |
|-----|---------|
| C1. GEMINI.md配置 | 全局规则 + 分层披露（领域规则、项目规则） |
| C2. Workflows工作流 | 高频/固定流程固化为 Workflow，串行执行 + turbo 跳过确认 |
| C3. MCP协议 🚧 | USB 接口类比，配置后 Agent 获得我的能力/权限 |
| C4. settings.json | 房子+钥匙类比，配置 MCP Server 的技术配置中心 |

## 核心类比汇总

| 概念 | 类比 |
|-----|------|
| Agent | 有边界的执行者 |
| 提示词 | 需求文档 |

## 学习亮点

1. **"脚疼砍脚"类比**（B1）：Agent 优化≠正确理解需求，需要明确需求+主动协作
2. **分层披露设计**（C1）：GEMINI.md → dev.md → 项目规则，按需加载节省上下文
3. **Artifact 残缺状态分析**（B4）：缺少 plan 就强行继续 = 在错误上叠加更多错误
4. **MCP 配置流程**（C3）：确保平台支持 + 创建用户 + 获取 Token + 配置 setting
