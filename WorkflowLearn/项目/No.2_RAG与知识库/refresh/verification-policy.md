# verification-policy

> 最近确认时间：2026-04-16

## 项目级规则

### 可直接迁入 `concepts.md` 的内容

- RAG 的基本架构与流程
- Embedding / 向量 / 语义检索的基本机制
- 数据质量、分块、检索错误与生成错误的通用判断框架
- RAG 适用边界的原则性判断

### 必须进入 `volatile.md` 的内容

- 具体模型、具体数据库、具体产品推荐
- 固定 chunk 参数和 Top-K 默认值
- 示例指标阈值
- 与某家框架或厂商绑定的操作建议

### 旧状态继承规则

- 旧“已掌握/已完成”不自动继承到新状态层
- 旧验证记录可以迁入 `labs.md`，但要先去除不可恢复的聊天碎片

### 必须回退的情况

- 结论建立在厂商快照或过期默认值上
- 无法区分这是检索问题、生成问题还是数据问题
- 概念与案例判断冲突
