## 核心可交付产物

- `RAG product spec`
  - 证明能力：能把知识库能力翻译成真实产品方案，而不只是后端架构图。
  - 可复用位置：作品集主案例、PRD、面试 case。

- `knowledge architecture map`
  - 证明能力：能解释离线与在线链路、数据层与回答层的分工。
  - 可复用位置：系统图、架构评审、团队对齐。

- `retrieval diagnosis SOP`
  - 证明能力：能区分数据问题、检索问题和生成问题。
  - 可复用位置：上线运维、失败复盘、面试深挖。

- `grounding / citation design`
  - 证明能力：能把引用、出处和 fallback 设计成用户体验的一部分。
  - 可复用位置：交互设计、信任设计、作品集附录。

- `RAG fit memo`
  - 证明能力：能判断什么时候该上 RAG，什么时候不该上。
  - 可复用位置：技术选型、产品评审、面试题。

- `C1 capstone pack`
  - 证明能力：能把产品 spec、eval plan 和 fallback / citation design 串成一个完整案例。
  - 可复用位置：重点作品、portfolio、中长篇案例讲解。

## 应体现的岗位能力

- 能判断什么时候该上 RAG，什么时候不该上。
- 能把知识库能力翻译成用户价值、引用体验和 fallback 机制。
- 能设计检索质量、回答质量和上线指标。
- 能理解 retrieval 在 agent 系统里扮演的 context / knowledge layer 角色。
