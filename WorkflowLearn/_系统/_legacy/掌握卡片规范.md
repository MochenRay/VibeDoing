# 掌握卡片规范（旧版迁移参考）

> 本文件不再是 `WorkflowLearnSystem` 的正式状态协议。
> 当前系统优先使用 `curriculum/concepts.md`、`curriculum/labs.md`、`state.md`、`handover.md`。
> 本文件只用于理解旧 `掌握卡片/*.md` 该如何拆分迁移。

## 文件命名

格式：`[序号]. [概念名].md`
示例：`A1. Agent执行模式.md`

---

## 旧版实时更新机制

> [!CAUTION]
> 这是 1.0 的旧规则，不再作为新版正式前提。
> 新版要求的是“长期证据必须写回”，但不再要求所有教学记录都实时追加到单一卡片文件。

**迁移映射**：
1. 稳定理解与定义 → `curriculum/concepts.md`
2. 错误、归因、挑战 → `curriculum/labs.md`
3. 时效项与待核实内容 → `curriculum/volatile.md`
4. 当前进度、复习状态、回退原因 → `state.md` / `handover.md`

## 卡片结构（必填/选填分离）

```markdown
# 掌握卡片：[序号]. [概念名]

## 🧠 我的理解 【必填】
（用用户自己的话描述概念，必须是用户原话）

## 📖 源真理定义 【必填】
> [引用原文的严谨定义]

## 🚫 我的错误记录 【必填，可为空】
（用户在学习过程中实际犯过的错误，有归因分析）
- **场景**：[在什么情况下犯错]
- **错误**：[错误描述]
- **归因**：[为什么会犯这个错—概念混淆/边界不清/旧知干扰...]
- **修正**：[正确做法]

---
以下为选填模块，根据对话内容按需添加
---

## 🔗 类比挂钩 【选填】
（记录对话中有效的类比）

## ⚠️ 常见误区 【选填】
（通用知识：大家都容易犯的错误）

## 💡 验证高光 【选填】
（实操验证中的亮点表现）

## 🧱 关联知识 【选填】
- **前置节点**：[[前置概念]]
- **后置节点**：[[后置概念]]

## 💪 实操挑战 【选填】
完成 `/实操` 微任务后解锁。
```

## 质量要求

### 必填模块（3个）
| 模块 | 要求 |
|------|------|
| 我的理解 | 用户原话，不是AI改写 |
| 源真理定义 | 必须有引用来源 |
| 我的错误记录 | 有错误时填写 + 归因；无错误则留空 |

### 选填模块（5个）
- 类比挂钩、常见误区、验证高光、关联知识、实操挑战
- 有对话内容时添加，没有则不添加
- 不强制每个模块都填写
