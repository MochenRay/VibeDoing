# Core-Learning

> 最近确认时间：2026-04-16

## 1. 目的

`Core-Learning` 定义的是 `WorkflowLearnSystem` 的宿主无关学习内核。
它只描述“学什么、何时算学过、何时必须回写、何时必须刷新”，不描述任何宿主专属工具名、命令入口或写文件方式。

## 2. 设计边界

- Core 只负责语义与流程。
- 教学行为合同单独放在 `Teaching-Contract`，不混进宿主适配层。
- Adapter 只负责把 Core 映射到 Claude、Gemini、Codex 的真实能力。
- Core 不引用任何具体工具名、slash command、终端命令或产品 UI。
- Core 不保存一次性运行态，不保存聊天转录，不保存宿主缓存。
- Core 只接受可重建、可验证、可回写的长期文档与状态。

## 3. 核心目标

1. 支持学习项目初始化。
2. 支持跨会话状态恢复。
3. 支持分阶段学习、验证与回写。
4. 支持时效知识刷新。
5. 支持把学习结果转成现实判断、方案和可展示成果材料。
6. 支持在学习中对快速演化领域做动态差异扫描，而不是把现有讲稿当成永远正确。

## 4. 核心概念

- `Project`：一个长期学习主题。
- `Module`：项目中的一个可独立学习单元。
- `State`：当前学习的可恢复状态。
- `Artifact`：学习后必须留下的长期结果。
- `Validation`：对理解、事实和产出的确认。
- `Refresh`：对时效内容的重新核实与更新。
- `Delta Scan`：在学习前或学习中，对当前外部生态是否出现新概念、新默认做法、新 SOP 或旧说法失效进行快速检查。
- `Teaching Contract`：定义教学阶段、回退、复习和收口规则。
- `Adapter`：宿主侧实现层，不进入核心规则。

## 5. 学习内核必须遵守的规则

- 任何学习都必须先恢复状态，再开始新工作。
- 任何教学都必须遵守显式阶段与回退门槛。
- 任何结论都必须可验证，不能只靠对话记忆。
- 任何可复用知识都必须落到长期文档。
- 任何时效性判断都必须显式标记最近确认时间。
- 任何验证失败都必须阻止写回为正式结论。
- 任何宿主差异都只能出现在 adapter，不得污染 core。
- 任何快速变化领域都允许并鼓励在学习中途触发 `Delta Scan`，发现新概念或旧结论漂移时先刷新再继续教学。

## 6. 标准循环

1. 恢复当前状态。
2. 选择本轮学习范围。
3. 产出学习内容。
4. 做验证与纠错。
5. 写回长期文档。
6. 检查是否需要差异扫描或刷新。
7. 更新下一步状态。

## 7. 产物要求

每一轮学习至少要留下以下内容中的一部分：

- 主题摘要
- 关键概念
- 验证结论
- 未解决问题
- 下一步动作
- 时效刷新需求
- 差异扫描发现的新概念、旧说法漂移或待补查项

如果本轮只是探索，没有形成可写回结论，也必须留下“探索记录 + 未确认项 + 下一步”。

## 8. 质量标准

- 内容要短，但不能空。
- 内容要可恢复，但不能依赖运行态。
- 内容要可验证，但不能把验证结果伪装成事实。
- 内容要可刷新，但不能把时效快照伪装成稳定知识。

## 9. 不要放进 Core 的东西

- Claude / Gemini / Codex 专属命令
- 具体写文件 API 或工具名
- 本地缓存路径
- 某宿主的 UI 操作步骤
- 一次性对话记录
- 临时调试说明

教学轮次、论述守门、回退阈值和复习计划不在 adapter 中定义，统一放在 `Teaching-Contract`。
