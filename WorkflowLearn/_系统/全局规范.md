## 目录边界

- `_系统/` 只放系统内核、schema、角色说明、用户画像与通用规范。
- `项目/` 放正式课程项目及其 `curriculum/`、`refresh/`、状态文件。
- `专题/` 放当前阶段的领域聚焦、岗位导向路线图、全景产出清单和专题分析归档。
- 不要把某个阶段的专题学习计划、岗位导向文档或领域分析直接放进 `_系统/`。

## 当前系统形态

`WorkflowLearn` 现在采用：

- `host-agnostic core`
- `host-specific adapters`
- `single-source canonical markdown`

换句话说：

- 学习语义、状态流转、schema 在当前仓库统一定义
- Codex / Claude / Gemini 只在 adapter 层解释宿主差异
- 不再依赖仓库外部目录才能运行

## 标准学习闭环

现行闭环固定为：

`恢复 -> 差异扫描 -> 学习 -> 验证 -> 写回 -> 刷新 -> 更新下一步状态`

如果本轮是教学型模块，则学习与验证内部还必须遵守：

`讲解 -> 理解确认 -> 掌握验证 -> 费曼/实操 -> 结算 -> 复习计划`

其中：

- 恢复失败，必须阻塞
- 验证失败，不能正式写回
- 时效项不确定，进入 `volatile` 或刷新队列
