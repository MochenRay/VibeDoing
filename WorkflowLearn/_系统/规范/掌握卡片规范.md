# 掌握卡片规范

## 文件命名

格式：`[序号]. [概念名].md`
示例：`01. Mission_Control.md`

## 卡片结构

```markdown
# 掌握卡片/[序号]. [概念名]

## 🧠 我的理解
（用用户自己的话描述概念）

## 🔗 类比挂钩
（记录对话中有效的类比）
**"[类比名]"**
- [类比内容]

## 🚫 踩过的坑
（记录用户犯过的错误及反思）
- **错误**：[错误描述]
- **原因**：[为什么会犯这个错]
- **教训**：[以后如何避免]

## 📖 源真理定义
> [引用原文的严谨定义]

## 🧱 关联知识
- **前置节点**：[[前置概念]]
- **后置节点**：[[后置概念]]

## 💪 实操挑战
完成 `/实操` 微任务后解锁考官验证。
```

## Frontmatter（验证通过后添加）

```yaml
---
status: mastered
mastered_date: 2026-01-09
review_count: 1
last_review: 2026-01-09
next_review: 2026-01-10
---
```

## 质量要求

1. **我的理解**：必须用用户原话，不是AI的标准定义
2. **踩过的坑**：必须有，没踩过坑说明学习不够深入
3. **实操挑战**：必须链接到实操Agent
