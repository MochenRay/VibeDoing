# 项目信息结构

项目信息.md 的标准字段定义。

---

## 📋 完整字段结构

```yaml
# 项目信息

## 基础信息
- 项目名：[项目名]
- 创建日期：[YYYY-MM-DD]
- 源真理版本：[版本或来源]

## 用户画像（引用全局或项目级）
- 用户身份：[超级个体/学生/...]
- 技术背景：[描述]
- 兴趣图谱：见 [[_系统/用户画像]] 或项目级定义

## 学习状态
- 当前Agent：[导师Agent/实操Agent/考官Agent/课程Agent]
- 已掌握节点：[列表]
- 当前节点：[节点名]
- 待解锁节点：[列表]
- 掌握程度：[百分比]

## 难度追踪
- 当前难度：[基础/进阶/挑战]
- 本节点得分历史：[7, 5, 8]
- 薄弱点标记：[概念A, 概念B]

## 错误记录
当前节点错误记录:
  - 概念: [概念名]
    实操错误次数: 0
    验证错误次数: 0
    变式题历史: []
    状态: [进行中/已通过/需重学]

## 前测结果
前测结果:
  熟悉度: [A/B/C/D]
  已掌握: [列表]
  待深入: [列表]
  跳过节点: [列表]

## 复习计划
已掌握节点复习:
  - 节点: [节点名]
    掌握日期: [YYYY-MM-DD]
    下次复习: [YYYY-MM-DD]
    复习间隔: [1天/3天/7天/...]
```

---

## 🔄 更新时机

| 事件 | 更新字段 |
|-----|---------|
| 进入新Agent | `当前Agent` |
| 回答问题 | `难度追踪.本节点得分历史` |
| 答错 | `错误记录.错误次数` |
| 节点通过 | `已掌握节点`、`当前节点`、`复习计划` |
| 全部完成 | `掌握程度` = 100% |

---

## 📍 各Agent职责

| Agent | 负责更新的字段 |
|-------|---------------|
| 课程Agent | 基础信息、待解锁节点、前测结果 |
| 导师Agent | 当前Agent、难度追踪 |
| 实操Agent | 错误记录.实操错误次数 |
| 考官Agent | 错误记录.验证错误次数、复习计划 |
| 报告Agent | 不更新，只读取 |
