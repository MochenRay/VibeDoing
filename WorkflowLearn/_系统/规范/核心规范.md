# 核心规范

## 项目信息模板

```markdown
# 项目信息

- **项目名**：[项目名]
- **创建日期**：[YYYY-MM-DD]
- **用户身份**：[职业/背景]
- **技术背景**：[描述]
- **学习目标**：
  - [目标1]
  - [目标2]
- **兴趣图谱** (用于隐喻)：
  - **核心**：[主要兴趣]
  - **辅助**：[次要兴趣]
- **当前状态**：
  - 当前Agent：[课程/导师/实操/考官]
  - 已掌握节点：[]
  - 当前节点：[节点名]
  - 待解锁节点：[]
  - 实操状态：[未开始/已通过]
  - 掌握程度：0%
- **源真理版本**：[版本描述]
```

## 文件操作规范

1. **路径规则**：
   - 文件操作：使用相对路径 `项目/`、`_系统/`
   - Obsidian路径：使用完整路径 `{WORK_DIR}/项目/...`

2. **目录结构**：
   ```
   vibe-learning/
   ├── GEMINI-Learning.md      # 主入口
   ├── _系统/
   │   ├── 导师Agent.md
   │   ├── 实操Agent.md

   │   ├── 课程Agent.md
   │   └── 规范/
   │       ├── 核心规范.md
   │       └── 掌握卡片规范.md
   └── 项目/
       └── [项目名]/
           ├── 项目信息.md
           ├── 学习路径.md
           ├── 源真理/
           └── 掌握卡片/
   ```

## 项目信息更新规范

以下字段需要在对应时机更新：

| 字段 | 更新时机 | 更新者 |
|-----|---------|-------|
| 当前Agent | Agent切换时 | 各Agent |
| 已掌握节点 | 实操Agent验证通过 | 实操Agent |
| 当前节点 | 开始新节点学习 | 导师Agent |
| 实操状态 | 实操通过 | 实操Agent |
| 掌握程度 | 节点完成时计算 | 课程Agent |
