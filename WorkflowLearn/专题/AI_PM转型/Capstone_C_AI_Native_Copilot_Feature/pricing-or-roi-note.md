# pricing-or-roi-note

## 1. 成本结构

- 文本与多模态推理调用
- 检索与数据源访问
- 审核与人工接管成本
- trace、日志和反馈数据基础设施

## 2. 价值锚点

- 缩短客户研究到初稿的时间
- 降低方案准备的人力重复劳动
- 提高可复用内容的沉淀率
- 降低无依据承诺带来的业务风险

## 3. 定价思路

- `seat + usage` 混合更合理
- 不建议只按 token 计费，因为用户买的是可交付效率，不是模型调用量

## 4. ROI 观察口径

- 单份方案平均准备时间下降
- 人工返工次数下降
- 试点团队周产能提升
- 高风险错误带来的损失避免
