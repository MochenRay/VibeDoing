# multimodal-flow

## 1. 输入

- 会议录音转写
- 客户需求文档或 PDF
- 截图或白板照片
- CRM 基础字段
- 历史中标方案

## 2. 处理流程

1. 解析文本、附件和截图中的关键信息
2. 抽取客户目标、限制条件、风险项和缺失信息
3. 从历史案例和知识库中检索相关片段
4. 生成结构化 brief 和方案初稿
5. 给每个关键段落附证据和风险标签
6. 将不可确认内容显式送入人工确认队列

## 3. 输出

- 结构化客户 brief
- 可编辑方案章节
- 缺失信息清单
- 风险提示与审核建议

## 4. 必须可验证的步骤

- 关键客户事实抽取
- 引用的来源映射
- 价格、交付承诺和高风险声明

## 5. 可以模糊生成的步骤

- 草稿行文
- 标题建议
- 次级措辞优化
