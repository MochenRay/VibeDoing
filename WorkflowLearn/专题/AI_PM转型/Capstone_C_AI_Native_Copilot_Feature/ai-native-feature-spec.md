# ai-native-feature-spec

> 默认样例：客户研究与方案撰写 Copilot

## 1. 问题

销售、客户成功和解决方案团队在做客户研究与方案撰写时，常见问题不是“信息完全没有”，而是：

- 信息散落在会议纪要、CRM、过往案例、报价规则和产品资料里
- 用户不需要一个会聊天的机器人，而需要一个能把信息直接编排进工作界面的 Copilot
- 当前最大摩擦是从资料收集到方案成稿之间的切换成本、证据丢失和反复人工改写

## 2. 为什么这里应该做 AI-native feature

- 目标不是回答一个问题，而是持续协助完成一个工作流中的产物
- 用户下一步动作是继续编辑、确认、补证据、发起审批，而不是结束对话
- 输出必须可复用进界面组件、文档块和行动列表，所以不能只回一段自然语言

## 3. 目标用户

- 解决方案经理
- 销售运营
- 客户成功经理

## 4. 核心任务

1. 汇总客户背景与会议纪要
2. 生成可编辑的客户 brief
3. 给出方案骨架、风险提示和缺失信息
4. 把高风险段落送进人工审核

## 5. 关键交互

- 用户上传会议纪要、文档或截图，或直接拉取 CRM / 历史案例
- 系统渲染结构化 brief、证据抽屉和待确认事项
- 用户在行内接受、修改、拒绝 AI 建议
- 系统把修改行为和显式反馈送回评估与优化链路

## 6. 核心成功指标

- `time to first usable draft`
- `accepted suggestion rate`
- `manual edit ratio`
- `unsupported claim rate`
- `review escalation rate`

## 7. 非目标

- 不追求完全自动发出对外承诺
- 不把价格、合同条款和高风险承诺自动放行
- 不把此功能做成独立聊天应用
