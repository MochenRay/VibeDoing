# case-study

## 1. 案例一句话

我把“客户研究与方案撰写”从一个分散、反复复制粘贴的工作流程，改造成了一个 AI-native Copilot workspace。

## 2. 为什么不是聊天框

这个场景的核心不是问答，而是把来源、草稿、证据和审核动作压到同一个工作界面里。
如果只用聊天框，用户仍然要自己搬运内容、确认风险并重新整理成产物。

## 3. 方案亮点

- 主 surface 采用 `workspace canvas + inline edit`
- 每个关键段落都绑定证据抽屉
- 用户修改和反馈会回流到 eval 数据集
- 高风险段落默认进入人工审核

## 4. 证明了什么

- 我能判断 AI-native feature 应该长什么样
- 我能把 surface、feedback、fallback 和 launch gate 一起设计进去
- 我能讨论上线和 ROI，而不只讲交互概念
