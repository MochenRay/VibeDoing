# fallback-trust-design

## 1. 核心原则

- 证据不足时不硬生成确定结论
- 高风险内容默认进入人工确认
- UI 上必须让用户一眼看出“可直接用”与“待核验”

## 2. 回退梯度

1. 优先补上下文
2. 若仍不足，降级成“待补信息草稿”
3. 高风险段落直接标记 `needs_human_review`
4. 工具失败时退回来源列表和人工输入表单

## 3. 高风险内容

- 价格与折扣承诺
- 法务或合规描述
- 交付时间保证
- 客户敏感信息的外发内容

## 4. 信任设计

- 每个关键段落都要能展开证据
- 没有证据时要明确显示“暂无可靠依据”
- 风险提示不能藏在二级页面
