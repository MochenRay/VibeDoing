# feedback-loop-blueprint

## 1. 显式反馈

- 段落级 `accept / edit / reject`
- “有帮助 / 无帮助”
- 风险标记纠正
- 对证据缺失的举报

## 2. 隐式反馈

- 首次草稿后被改写的字数和位置
- 哪类建议最常被删除
- 哪类段落最常触发人工审核
- 哪些工具调用经常失败或被重新执行

## 3. 数据回流路径

`产品事件 -> trace / artifact -> 样本归档 -> eval 数据集 -> prompt / schema / UI 调整`

## 4. 节奏

- 周节奏：看在线指标和失败样本
- 双周节奏：更新评估集
- 月节奏：决定是否改 UI、改 schema、改策略

## 5. 负责人

- 产品：定义反馈优先级和迭代方向
- 设计：看 surface 和交互摩擦
- 工程 / ML：看提示、schema、工具链和数据质量
