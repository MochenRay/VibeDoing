# Presentation Script

> 最近确认时间：2026-04-16
> 用途：5-8 分钟讲解稿

## 开场

这不是一个“给政策文档加聊天框”的项目，而是一套 `Grounded Knowledge Assistant`。
我解决的核心问题是：在高风险政策问答里，系统不只要会回答，还必须会举证、会收窄、会转人工。

## 问题

城市治理和政务服务里，用户常会问：

- 外地人能申请公租房吗
- 某个流程有没有例外条件
- 当前版本政策和旧版本有什么区别

这里最大的问题不是没文档，而是：

- 用户说法和文档说法不一样
- 知识分散
- 文档会更新
- 用户必须知道答案依据是什么

## 为什么不是纯模型

纯模型最大的问题是：

- 不保证基于最新知识
- 不能天然给出处
- 高风险场景里容易“看起来很自信”

所以这里必须做 grounded answer，而不是普通聊天回答。

## 我的方案

我把系统拆成三层：

- data layer
- answer layer
- trust layer

其中 trust layer 是关键，因为它决定：

- 引用怎么展示
- 什么时候提示版本冲突
- 什么时候只能收窄回答
- 什么时候必须转人工

## 我怎么判断它能不能上线

我专门做了三层 eval：

- retrieval layer：有没有找到对的依据
- answer layer：回答是否忠实于依据
- trust layer：引用是否可追溯，证据不足时有没有正确 fallback

也就是说，我不是只做了个 demo，而是已经把它推进到可以讨论 launch gate 的状态。
