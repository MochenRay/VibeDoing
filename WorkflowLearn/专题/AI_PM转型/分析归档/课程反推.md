# AI 应用 PM 与 Agent/Workflow PM 课程反推

更新时间：2026-04-16

## 1. 目标

这份文档不回答“哪些知识稳定”，而回答更实际的问题：

- 如果目标岗位是 **AI 应用 PM**
- 或者目标岗位是 **Agent / Workflow PM**

那应该学什么，不该学什么，以及现有课程要怎么改。

## 3. 两类岗位的区别

### 3.1 AI 应用 PM

这类岗位关心的是：

- 某个 AI 功能是否能带来用户价值
- 是否能稳定上线
- 是否能被评估、回滚、迭代
- 是否能嵌进真实业务流程

它的典型对象是：

- AI 搜索
- 对话式分析
- 智能摘要
- 工作流 Copilot
- 垂直场景智能助手

它不是在做“Agent runtime 本身”，而是在做 **AI 功能产品**。

### 3.2 Agent / Workflow PM

这类岗位关心的是：

- agent 怎么拆角色
- workflow 怎么编排
- tool / API / state / memory 怎么管理
- eval / telemetry / tracing / handoff / rollback 怎么做
- 系统如何在生产环境可靠运行

它的典型对象是：

- 多步任务 agent
- 企业 workflow builder
- AI agent platform
- deployment / orchestration / observability / governance

它不是只做一个 AI feature，而是在做 **AI runtime / workflow system / agent operating layer**。

## 4. 从 JD 反推能力模型

### 4.1 AI 应用 PM 的核心能力

从 Rezolve、AHEAD、Arbital、Doxel 这类 JD 往回看，能力要求主要集中在：

1. **问题定义与价值判断**
   - 能把“这里加 AI”翻译成清晰的用户问题、业务目标和 success metrics

2. **AI 方案设计**
   - 知道什么时候用 prompt
   - 什么时候用 retrieval
   - 什么时候需要 workflow
   - 什么时候必须加 fallback / human review

3. **质量与上线判断**
   - 会做 eval
   - 会看 hallucination / latency / task success / cost / safety
   - 会做 go / no-go 决策

4. **AI feature 的产品化**
   - 能把能力嵌进产品 flow
   - 能定义 guardrails、约束、输出格式、解释方式

5. **跨团队推动**
   - 和工程、设计、算法、业务一起做 trade-off

### 4.2 Agent / Workflow PM 的核心能力

从 Netomi、Mistral、Vanta、Turgon 这类 JD 往回看，能力要求主要集中在：

1. **Agent / Workflow 拆解能力**
   - 角色分工
   - task decomposition
   - tool boundaries
   - orchestration logic

2. **状态与运行时理解**
   - memory / context / state
   - long-horizon tasks
   - handoff
   - failure recovery

3. **生产级评测与观测**
   - tracing
   - telemetry
   - quality gates
   - regression
   - drift

4. **协议与接口能力**
   - APIs
   - tools
   - MCP
   - connectors
   - contracts

5. **平台化思维**
   - 可配置
   - 可治理
   - 可复用
   - 可部署
   - 可审计

## 5. 课程反推结论

### 5.1 如果目标是 AI 应用 PM

优先级最高的课程应该是：

1. `No.1` 精简版
   - 只保留底层机制
2. `No.2`
   - Retrieval / RAG / knowledge grounding
3. `No.4`
   - Prompt / schema / tool use / workflow basics
4. `No.3`
   - Evals / metrics / experiments / rollout
5. `No.5`
   - Fallback / confidence / HITL / expectation management
6. `No.6` 精简版
   - 只补 agent 概念和“何时该上 agent”

### 5.2 如果目标是 Agent / Workflow PM

优先级最高的课程应该是：

1. `No.6`
   - Agent / MCP / orchestration / multi-agent / workflow
2. `No.4`
   - Prompt → Tools → Structured Outputs → Tool contracts
3. `No.3`
   - Agent/workflow evals / observability / quality gates
4. `No.2`
   - Retrieval 作为 agent context / memory / knowledge layer
5. `No.1` 精简版
   - 理解 context / hallucination / sampling limits
6. `No.5` 精简版
   - 人机协同和 escalation 边界

## 6. 现有 No.1-No.6 怎么改

### 6.1 No.1 大模型基础认知

保留：

- Token
- Context
- Temperature / sampling
- Hallucination
- 下一个 token 预测的本质

删除或移出主线：

- 主流模型对比表
- 价格表
- 静态选型建议

定位：

- 两条岗位路线都需要
- 但只能当“基础语法”，不能当主菜

### 6.2 No.2 RAG 与知识库

保留：

- RAG 流程
- Embedding / ANN / Chunking
- 检索错误 vs 生成错误
- 适用边界

增强：

- reranking
- query transformation
- retrieval eval
- grounding / source attribution
- agent 使用 retrieval 的模式

定位：

- AI 应用 PM：核心必修
- Agent / Workflow PM：重要配套

### 6.3 No.3 AI 产品评估指标

保留：

- 指标和实验基本框架

重写重点：

- 从“分类/性能指标课”升级成“AI / Agent eval 课”
- 补：
  - task success
  - tool selection accuracy
  - argument correctness
  - handoff accuracy
  - regression
  - drift
  - human eval
  - LLM-as-judge

定位：

- 两条岗位路线都必须强化

### 6.4 No.4 Prompt Engineering

保留：

- prompt structure
- system prompt
- few-shot
- decomposition

重写重点：

- 强化：
  - function calling
  - structured outputs
  - prompt + tool contracts
  - prompt + eval loop
  - prompt 不是 feature，本质上是 behavior spec

定位：

- AI 应用 PM：必修
- Agent / Workflow PM：更必修

### 6.5 No.5 AI 产品设计特殊性

保留：

- fallback
- confidence
- human-in-the-loop
- expectation management
- error feedback

建议：

- 不再作为独立大模块
- 并入“AI PM 产品化设计”主线

定位：

- AI 应用 PM：有用
- Agent / Workflow PM：辅助知识

### 6.6 No.6 Agent 概念

保留：

- agent vs 普通对话
- ReAct
- tool design
- MCP
- multi-agent

重写重点：

- 从“概念介绍”升级成：
  - workflow vs agent
  - single-agent vs multi-agent
  - state / memory / progress artifacts
  - runtime boundaries
  - orchestration contracts
  - observability
  - evaluation

定位：

- Agent / Workflow PM：主修核心
- AI 应用 PM：选修但需要精简版

## 7. 新课程地图

### 7.1 AI 应用 PM 路线

建议课程主线：

1. `LLM Fundamentals for PM`
2. `Retrieval and Grounded AI Products`
3. `Prompt, Schema, and Tool-Enabled UX`
4. `AI Quality, Evaluation, and Launch Decisions`
5. `Fallback, Trust, and Human-in-the-Loop Design`
6. `Applied AI Product Cases`

对应产出：

- 一个 AI feature PRD
- 一套 eval plan
- 一个 launch / rollback 决策框架
- 两个垂直场景案例拆解

### 7.2 Agent / Workflow PM 路线

建议课程主线：

1. `Agent Fundamentals and Workflow Boundaries`
2. `Tool Contracts, MCP, and Connectors`
3. `State, Memory, and Long-Running Tasks`
4. `Orchestration and Multi-Agent Patterns`
5. `Agent Evals, Observability, and Quality Gates`
6. `Workflow Builder / Agent Platform Cases`

对应产出：

- 一个 agent workflow design doc
- 一个 multi-agent orchestration spec
- 一套 agent eval matrix
- 一个 runtime / telemetry / failure-handling 框架

## 8. 你最该补的能力

如果目标是尽快转向这两类岗位，你最需要补的不是“再背一遍模型表”，而是下面 4 类能力：

1. **把 AI 问题拆成可执行系统**
   - prompt
   - retrieval
   - tool
   - workflow
   - HITL

2. **把 AI 能力转成可上线产品**
   - success criteria
   - fallback
   - quality gates
   - rollout / rollback

3. **把 AI 系统做成可评估对象**
   - representative samples
   - ground truth
   - tool eval
   - workflow eval
   - agent handoff eval

4. **把复杂系统讲清楚**
   - 为什么这么设计
   - 为什么不选另一个方案
   - 风险在哪里
   - 怎样验证

## 9. 最终结论

### 9.1 对 AI 应用 PM

最重要的是：

- `No.2`
- `No.4`
- `No.3`
- `No.5`

`No.1` 只保留精简基础。
`No.6` 只保留“何时应该用 agent”的部分。

### 9.2 对 Agent / Workflow PM

最重要的是：

- `No.6`
- `No.4`
- `No.3`
- `No.2`

`No.1` 作为基础。
`No.5` 只保留 escalation / trust / HITL 的部分。

### 9.3 对现有仓库的直接动作

建议后续这么改：

1. 把 `No.1` 拆成 `基础机制` 和 `时效快照`
2. 把 `No.3` 重写成 `AI / Agent Evals`
3. 把 `No.4` 重写成 `Prompt + Tools + Structured Outputs + Policies`
4. 把 `No.5` 降级为产品化补充模块
5. 把 `No.6` 升级为 `Agent / Workflow / Harness Engineering`

如果只保留一条最有竞争力的路线，我建议优先做：

**Agent / Workflow PM 主线 + AI 应用 PM 产品化能力补全**

因为这条组合最能同时覆盖：

- 企业级 workflow automation
- AI agents
- production readiness
- evals
- PM 的产品落地能力
